内存分配与映射

Posted on 2017-06-15 | In 编程 | Views:

内存分配与映射的一些知识总结。

内存分配与映射

内存分配

在简单的字符设备一文的知识点补充中，已经总结了kmalloc和vmalloc的原型和基本用法，下面深入的总结一下。

用户态内存分配

malloc/free

  • 虚拟地址空间
  • 返回的是线性地址
  • 在驱动程序中不能使用

内核态内存分配

包括以下三种：

  • kmalloc
  • vmalloc
  • __get_free_page及__get_free_pages
  • alloc_page及alloc_pages

特别说明，后两类函数返回的是struct page结构体，其它函数一般返回分配内存的首地址（虚拟地址）。

Kmalloc

原型已经总结过，见原来的博文。

下面再介绍一下参数：

size参数

内核负责管理物理内存，物理内存只能按页面进行分配；
Linux处理内存分配的方法是，创建一系列内存对象池，每个池中的内存块大小是固定一致的。处理分配请求时就直接在包含有足够大的内存池中传递一整块给请求者；
size的大小一般是2的n次方，kmalloc能分配的最小内存块是32或64；
kmalloc能分配的内存块大小存在一个上限，为了保证代码的可移植性，不应利用kmalloc分配超过128KB的内存。

flags参数

最常用的flags参数是GFPKERNEL，表示内存分配是运行在内核空间的进程执行的；GFP前缀意味着调用get_free_pages来实现实际的内存分配。

flags标志主要有三类：（GFP 标志都在 <linux/gfp.h> 中定义 。）

  1. 行为标志 ：控制分配内存时，分配器的一些行为

  2. 区标志 ：控制内存分配在那个区(ZONE_DMA, ZONE_NORMAL, ZONE_HIGHMEM 之类)

  3. 类型标志 ：由上面2种标志组合而成的一些常用的场景

类型标志是最常用的，比如GFP_ATOMIC用于在中断处理例程或其它运行于进程上下文之外的代码中分配内存，不会休眠。GFP_KERNEL是内核内存的通常分配方法，可能会休眠。kmalloc允许空闲内存不足时进程进入休眠状态，因此使用GFP_KERNEL的函数必须是可重入的，GFP_KERNEL标志不能在中断处理例程中调用。

行为标志也有很多，都以__GFP开头，详情看源码，不在赘述。

区标志总结如下表：

__GFP_DMA 该标志请求分配发生在可进行DMA的内存区段中
区标志 描述
__GFP_HIGHMEM 该标志表明要分配的内存可位于高端内存
__GFP_DMA32 与1类似，只是只能被32位设备访问

Linux内核把内存分为三个区段：可用于DMA的内存、常规内存、高端内存。

针对常用情形总结如下~

场景 相应标志
进程上下文，可以睡眠 使用 GFP_KERNEL
进程上下文，不可以睡眠 使用 GFP_ATOMIC，在睡眠之前或之后以 GFP_KERNEL 执行内存分配
中断处理程序 使用 GFP_ATOMIC
软中断 使用 GFP_ATOMIC
tasklet 使用 GFP_ATOMIC
需要用于DMA的内存，可以睡眠 使用 (GFP_DMA｜GFP_KERNEL)
需要用于DMA的内存，不可以睡眠 使用 (GFP_DMA｜GFP_ATOMIC)，或者在睡眠之前执行内存分配

vmalloc

vmalloc分配虚拟地址空间的连续区域，但不保证物理地址空间的连续性；

vmalloc分配的地址空间在VMALLOC_START到VMALLOC_END的范围中；

vmalloc不能在原子上下文中使用，可能会引起休眠；

vmalloc申请的地址空间由vfree进行释放。

以下的内存分配函数都是宏定义。见<linux/gfp.h>头文件。

__get_free_page

补充一点，页的大小与体系结构有关，在 x86 结构中一般是32机 4KB或者64位机8KB。

alloc_page

这两个函数返回的是物理页面的结构体，下面会提到它。

内存映射

page结构体

首先介绍一下页的概念。对系统中每个物理页，都有一个page结构相对应。其头文件为<linux/mm_types.h>。

物理内存的每个页都有一个对应的 page 结构，看似会在管理上浪费很多内存，其实细细算来并没有多少。因为 里面有大量的共同体，我水平还不够，唉，看不懂:cry:。

于是，内核维护了一个或多个page结构数组，用于跟踪系统中的物理内存。

地址转换

内核提供了一些函数和宏用于在虚拟地址和物理地址之间进行转换。

VMA

虚拟内存区（VMA）用于管理进程地址空间中不同区域的内核数据结构。一个VMA表示在进程的虚拟内存中的一个同类区域。

Linux内核中，关于虚存管理的最基本的管理单元应该是struct vm_area_struct了，它描述的是一段连续的、具有相同访问属性的虚存空间，该虚存空间的大小为物理内存页面的整数倍。

struct vm_operations_struct 结构体用来处理进程的内存需求。

mmap

mmap方法是file_operations结构的一部分，并且执行mmap系统调用时将调用该方法，实现物理地址空间和虚拟地址空间间的内存映射。

用户态下文件操作的原型为：

mmap通过两种方式完成页表的建立，即完成物理地址和虚拟地址之间的映射：

  • 使用remap_pfn_range一次建立所有页表;
  • 使用nopage VMA方法每次建立一个页表，当用户访问VMA中的页，而该页又不在内存中时，将调用相关的nopage函数实现内存映射。

remap_pfn_range

该函数为addr~addr+size之间的虚拟地址构造页表。pfn（Page Fram Number）是虚拟地址应该映射到的物理地址的页面号，实际上就是物理地址右移PAGE_SHIFT位。
