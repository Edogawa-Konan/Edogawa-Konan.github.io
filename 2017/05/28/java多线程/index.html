创建多线程的方式

继承Thread类

​ 第一种是最简单的，只需要继承Thread类，并重写run方法即可，这个方法即时线程执行体。然后，创建实例调用start方法来让线程处于就绪状态。

这种方式主要问题在于，多个线程之间不能共享线程类的实例变量，因为它们是不同的实例。

getName可以得到线程的名字，setName可以自己给线程命名。

这里需要说明一下，java中main方法也是一个单独的线程，或者说叫主线程。所以这里实际上是三个线程。另外，线程必须用start调用使其处于就绪状态，如果直接调用run方法，那就会当做普通方法了。

实现Runnable接口

​ 第二种方式就麻烦一些了，需要定义Runnable接口的实现类，然后重写该接口的run方法，该接口是函数式接口。同样，run方法的方法体就是线程执行的方法体。然后，创建实例，作为Thread的构造器参数，即由Thread执行。最后调用Thread实例的start方法即可。

后台线程

​ 前面讨论的都是普通的线程，还有一种线程主要是为其它线程提供服务，称作守护线程。

其特点在于，前台线程全部死亡时，后台线程自动死亡，JVM的垃圾回收器就是典型的守护线程。

通过Thread的setDaemon(true)可以设置，但必须在start之前调用。

join、sleep和yield

join让一个线程等待另一个线程，通常有以下两种形式：

  1. join()等待被join的线程执行完
  2. join(long millis)等待被join的线程执行millis毫秒，如果还没有结束，就不再等待

sleep(long millis)让当前正在执行的线程暂停一段时间，并进入阻塞状态，它是Thread中的静态方法。调用这个方法之后，线程会进入休眠，在此期间不会得到执行的机会，常常用于暂停程序执行。

yield()方法也是Thread中的一个静态方法，它会让调用它的线程重新转入就绪状态，即让线程调度器再次调度一次，也就是说让优先级比它更高或者同级的线程有执行机会。当然，完全有可能还是执行当前线程。在多cpu的情况下，效果可能不明显。

注意：sleep抛出了InterruptedException，而yield没有抛出任何异常。
