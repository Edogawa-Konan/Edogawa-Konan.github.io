1111[Dijkstra+DFS]

Online Map

题意很简单，就是找出最短路径和时间最短的路径。

思路：利用两次Dijkstra和两次DFS即可，都是从终点DFS到起点，注意，距离的DFS可以简化，在保证路径最短的前提下，用weight这个数组保存到某个地点的时间。DFS之后对比输出即可。这是目前代码量最多的一道题！

1112[map和set的应用]

Stucked Keyboard

思路：用个map统计卡住的字符和其发现顺序index，然后Nobroken保存完好的键位。注意，只有重复次数%K==0才认为是坏的键位！之后，把map中字符按照index的顺序输出即可。

1113[超简单排序]

Integer Set Partition

思路：显然集合个数差要么是0要么是1，对原序列从小到大排序，然后下标0~N/2-1属于第一个集合，不论N是奇数还是偶数，中间那个必然归给集合2.

1114[并查集(贼复杂)]

Family Property类似题目1107，怎么越来越难了感觉。这还咋玩啊

思路：依旧是并查集的思路，只不过细节太多，处理起来就比较麻烦。细心细心！！

1115[BST树的建立及DFS]

Counting Nodes in a BST

思路：建立树的过程就是标准的递归过程，模板化了已经。这里要输出最后两层的结点数，不要用BFS了，麻烦，针对统计叶子节点个数、层的问题，上DFS吧。

1116[素数/hash]

Come on! Let’s C

思路：先把素数筛出来打表，然后根据输入建立hash，查询即可。

1117[智力问题]

Eddington Number

题意就是从一个序列中找到一个最大的k，要求这个序列中大于k的数的个数为k个。

思路：被自己的智商蠢哭了。想着用基数排序的思路，但是好麻烦且也没过全部用例。其实，只要从大到小排序，满足s[i]>i就继续循环，最后输出i即可。（不能从小到大排序然后从后面开始遍历，因为这样最后不一定满足s[i]>i，输入用例就是如此！）

1118[并查集]

Birds in Forest

来啊，继续出并查集啊，上瘾了。。

思路：都是套路啊，没什么特别的。

1119[已知先序后序求中序]

Pre- and Post-order Traversals

是的，题目就是这么变态。

思路：还是以前那种套路，不过如果树出现全偏向一边，会出现歧义，即是左孩子还是右孩子的问题。所以，这时候题目说输出一个就行，那么就默认右孩子了。这里根据后序的最后一个节点的前一个结点（根的右孩子）为基准，在先序中找到它，如果它紧挨着pre_b，那么显然就是上述提到的歧义了，否则就划分左右子树即可！

1120[set的运用]-E

Friend Numbers

思路：直接输入字符串，统计每个数字的各位和插入set即可。
